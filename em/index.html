Em

General Syntax & Usage

According to the HTML Standard, the <em> element represents stressed emphasis of its contents. The <em> element stacks, so if you have an entire phrase that is emphasized but would like to emphasize part of that phrase even more, you can use another <em> element inside. Though, it is advised not to overuse the <em> element. Here are some examples

<p>Dogs are my <em>favorite</em> animals.</p>
<p><em>Dogs are my <em>favorite</em> animals.</em></p>

Properties

Inline: The <em> element is an inline element, which means that it continues the flow of content on a line. Unlike block elements, inline elements don't start with a new line an you cannot set their width or height.

Flow Content: The <em> element belongs to the flow content category of elements which typically contain text or embedded content.

Phrasing Content: The <em> element belongs to the phrasing content category of elements which are flow elements that could belong inside a paragraph.

Palpable Content: The <em> element belongs to the palpable content category of elements which are simply elements that are visually rendered.

Accessibility

Although most every browser renders the <em> element as italicised, this effect can quite easily be replicated with a <span> element and some CSS, that is just a logical rendering that is outside the purpose of the element. When using the <span> element to italicize text you would like to emphasize instead of using the <em> element, you strip that semantic meaning so screen readers and bots will just see it as text of the same importance of whats around it. For visually impaired people who rely on these screen readers, they also lose that emphasis which can sometimes change the entire contextual meaning of the sentence. Consider the following example:

The man did not steal the money


The man did not steal the money

In the first example it is implied that while the man did not steal the money persay, he is still involved with the money in some capacity. In the second example, it is a simple denial of involvement altogether. This makes the man seem much less guilty of the crime. Without the emphasized text, that implied subtle meaning of the first example is lost and it comes across just as the second example.

Misuse

Since the <em> element usually renders as italicised text in the browser, it is often conflated with a styling tool to italicise any text you like, regardless of context. The <em> element has inherent semantic meaning, as stated above, that being to emphasize its text from the text around it.

Trivia

The <em> element has been around almost as long as HTML has, having been introduced in HTML 1.2 around 1994 and then standardized in HTML 2.0 around 1995.
