General Syntax & Usage

According to the HTML Standard, the <a> element represents a hyperlink (also referred to as a hypertext anchor) labeled by its contents.

In most use cases it will have an href attribute, with the target and title attributes occassionally appearing. Here is an example of one being used:

<a href="https://google.com/" target="_blank">Google</a>

Attributes

href:
Short for hypertext reference, this attribute specifies the destination of the linked resources. Within an <a> element this attribute can take:
  • Sections of a page with fragment URLs (Such as an element's ID prefixed with #)
  • Pieces of media files with media fragments
  • Telephone numbers with tel: prefixed URLs
  • Email addresses with mailto: prefixed URLs
target:
Determines which browsing context (tab, window, or frame) the hyperlink will open in. By default, putting a plain name in there will look for a frame with that name, so special keywords were allocated that start with an underscore to denote generic browsing contexts.
_self
will open the link in the current tab

_blank
will usually open the link in a new tab, but some user's might have it set to open in a new window.

_parent
will open in the parent browsing context. For example, if an <iframe> is embedded in a webpage and a user clicks on a link from within it with the target is set to _parent, then the link will open in the outer webpage that the <iframe> is embedded in.

_top
will open in the top most parent browsing context. Similar to _parent, except if there are multiple nested <iframe>s within each other and a link with _top is clicked in the innermost <iframe>, then the entire tab will change to that hyperlink instead of just that innermost frame.
download:
When this attribute is present the resource specified with href will be downloaded when the link is clicked. If a value is specified (which is optional), then that value will be used as the filename upon download.

ping:
This attribute is used to send POST requests to a space separated list of URLs when the link is used. These requests don't contain any other information in their body other than the word PING. They are particularly handy for tracking purposes.

rel:
This attribute defines the relationship between the linked resource and the current document. Here are the various values that rel can take
alternate
The link is an alternate version of this document, often used in conjunction with lang to provide an alternate language.

archives:
The linked document is related and is of historical interest.

bookmark:
The linked document serves as a bookmark.

external:
The linked document is not a local resource (to the organization or server).

first:
The link is a reference to the first document in a collection.

help:
The link references a help document for either the current document or the site as a whole.

index:
The link references a page that provdes an index for the current document.

last:
The link is a reference to the last document in a collection.

license:
The link is a reference to the legal or copyright information for the current document.

next:
The link is a reference to the next document in a collection.

nofollow:
This indicates that the link shouldn't be automatically followed by robots.

noreferrer:
This value indicates that browsers shouldn't send the referer header when the link is followed.

prev:
The link is a reference to the previous document in a collection.

search:
This value indicates that the link references a search facility in a site.

sidebar:
This value specifies a URL that should be displayed in a browser sidebar.

tag:
This value specifies a tag that applies to the document.

up:
This value provides a link to the document or section "up" from the current document, usually the parent or index document for the current URL.
hreflang:
Indicates the language of the linked resource, should match the core lang attribute on the linked document.

type:
The MIME type of the linked resource, typically just advisory information.

referrerpolicy:
The referer is usually the entire current URL of origin site. This includes the protocol, subdomain, domain, path, and query string. This referrerpolicy allows you to modify how much or how little of this URL to include, and to whom this URL is sent to.

Properties

Inline:
The <a> element is an inline element, which means that it continues the flow of content on a line. Unlike block elements, inline elements don't start with a new line an you cannot set their width or height.

Flow Content:
The <a> element belongs to the flow content category of elements which typically contain text or embedded content.

Phrasing Content:
The <a> element belongs to the phrasing content category of elements which are flow elements that could belong inside a paragraph.

Interactive Content:
The <a> element belongs to the interactive content category of elements which are, as the name implies, elements that the user can interact with. In this case, the user can click on the <a> element to follow/download the link.

Palpable Content:
The <a> element belongs to the palpable content category of elements which are simply elements that are visually rendered.

Interactivity

The <a> element has a few attributes that add various bits of interaction. Here are two of note:

The target element, as described above, allows you to specify which browsing context you would like the link to open up to. This is handy as its an easy built in way to create auxillary pop-up windows or new tabs so as to not leave the main browsing context entirely. One must be careful though to notify the user in some way what action will occur so as to not surprise their expectations too much. A simple (new tab) or (popup) phrase at the end of the link text usually will suffice.

The download attribute allows the user to immediately download the linked resource after clicking the link instead of having to open it before downloading. While a user might appreciate saving the click or two, this one must also be used carefully as users generally dislike having somethine download unexpectedly. Just as before, a [download] or (download) warning on the link will usually suffice.

Accessibility

While the <a> element can be styled with CSS just like any other element, it is recommended to not stray too far from the average look of a link. Often times people with vision difficulty rely on these link cues to determine what is an isn't a link. The WCAG guidelines lay out a pretty good baseline of what to follow if you're unsure of the styling of your links, check them out below.

Link Text

A link should always make sense out of context on its own, so avoid generic phrases such as:

  • click here
  • here
  • more
  • read more
  • link to [some desc]
  • info

Links should also be the right length. A link that's too long might be frustrating for mobile users or visually impaired people who have to listen to the entire link. Alternatively, a link that's too short or close together might prove difficult to click, even more so on mobile with imprecise fingers.

Link Styling

Whether you consciously think about it or not, you already have an idea of what a link looks like in your head. They're usually blue (specifically either #0645AD or #3366BB), have an underline, and the underline disappears when you hover on it. Utilizing these trends can help you massively in terms of users instantly understanding how to navigate your website. Though often times designers will want to help style their links to fit in with their website more than just the default style, so here are the WCAG guidelines for what you should and shouldn't change:

  • It is advised to keep the link's underline by default when unhovered (due to color blindness)
  • If it isn't underlined by default, it's recommended that:
    • The link have a 3:1 contrast ratio from the surrounding non-link text
    • The link presents a "non-color designator" on both mouse hover and keyboard focus
  • Other visual affects for hovering, such as background glows, drop shadows, and color changes also help users know that the item can be clicked on
  • Nearly any time a:hover are defined in CSS, it should be modified to be a:hover, a:focus so that the same visual link presentation is available when keyboard users navigate or 'tab' to the link.

Internationalization

Similar to most elements, the <a> element has an attribute to help determine its language, although it is slightly different than others. The <a> element has the attribute hreflang which specifies the language of the resource it is linking to, rather than the language of the element itself.

Misuse

The purpose of a link is to create a hyperlink to web pages, files, email addresses, locations in the same page, or another URL addressable piece of content. Without knowing this, it is easy to assume that a link is akin to a button without a background, as it is an element you click on in which an action occurs.

It is important to use the elements as they were intended though since it starts to raise issues for screen readers or other bots crawling through the website. For screen readers, they might read out your "button" as a link, confusing the user as to what its function is. For bots, they might try to follow the link so as to index the page, but wouldn't go anywhere so they would mark that that element as a dead link potentially hurting your SEO. A good general rule of thumb is that if you are using JavaScript with your <a> element, it is probably being misused.

Below are two elements that stylistically look the same, but one (the <a> element) functions acts as a link and one (the <button> element) as a button.

<a href="https://google.com">Google</a>
Google 
<button onclick="alert('You pressed the Google button');">Google</button>
