Ordered Lists (<ol>)

Overview

The <ol> element creates an ordered list of items. It can contain zero or more <li> elements. Ordered lists are used when the order of the elements is important to the semantic structure of the page. If the order of the elements does not matter/affect the structure of the document, an unordered list (<ul>) should be used instead.

An ordered list has a preceding marker before each list item, usually rendered as a decimal number. These markers can be changed in the HTML using the type attribute, or via CSS using the list-style-type property. Generally, defer to using the CSS list-style-type property unless the type adds semantic meaning to the document.

The permitted attributes are the global attributes, as well as three element specific attributes: reversed, start, and type.

Uses

The <ol> element should be used when the order of items in a list has inherent meaning. For example, ordered lists should be used to display instructions (such as a how-to guide or recipe) or a page's table of contents. Below, see an example of a table of contents.

Note that nesting ordered lists does not nest the markers in an "outline" style. However, this can be changed with CSS.

The type attribute sets the number style for the list. The permitted values are a for lowercase letters, A for uppercase letters, i for lowercase Roman numerals, I for uppercase Roman numerals, and 1 for numbers. Unless the type matters to the structure of the document, the CSS property list-style-type should be used instead.

CSS Tricks

Most of the styling with an ordered list comes from the styling of the markers used. Aside from the ability to change the marker using the type attribute or list-style-type property, you can use the CSS ::before selector with a background color and radius to create more custom markers. Use list-style: none on the <ol>, then for each <li> you can use li::before { content: x } for any x marker - this could be a special character, an image, or an emoji!
