Select

Syntax

According to developer.mozilla.org, the <select> element represents a control that provides a menu of options. This allows the user to easily give textual input without you having to format the response or control for responses outside the scope of the question. To use the <select> element, it must be used in conjunction with the <option> element. To create choices to select from in the <select> element dropdown menu, you simply add an <option> element as a child for each option. Here is an example: 

<select>
  <option value="vanilla">Vanilla</option>
  <option value="chocolate">Chocolate</option>
  <option value="strawberry">Strawberry</option>
  <option value="coffee">Coffee</option>
</select>

The <select> element also has many helpful attributes such as the multiple attribute which lets users select more than one option at once, and the size attribute which is used to specify how many options should be shown at once. Since it the most common use of the <select> element is in a form, it is generally good practice to add a name and id attribute so a label can be attached and the data can be passed back upon form submission.

One more common element you can use inside <select> is the <optgroup> element. This allows you to group similar <option> elements together. Here's an example of what that might look like

<select>
  <optgroup label="Ice Cream Flavors">
    <option value="vanilla">Vanilla</option>
    <option value="chocolate">Chocolate</option>
    <option value="strawberry">Strawberry</option>
    <option value="coffee">Coffee</option>
  </optgroup>
  <optgroup label="Sorbet Flavors">
    <option value="mango">Mango</option>
    <option value="peach">Peach</option>
    <option value="raspberry">Raspberry</option>
  </optgroup>
</select>

Applications

As was said above, the most common application for the <select> element is to gather user input for a question that requires a textual answer. A complete form and form submission for the <select> element might look like this

<form action="https://httpbin.org/post" method="POST">
  <label for="favAnimal">Choose your favorite animal:</label>
  <br>
  <select name="favAnimal" id="favAnimal">
    <option value="tiger">Tiger</option>
    <option value="lion">Lion</option>
    <option value="zebra">Zebra</option>
    <option value="hippo">Hippo</option>
  </select>
  <br>
  <input type="submit">
</form>

Interactivity

Since the <select> element is a <form> element, it is one of the few inherently interactive HTML elements. There are a few attributes that the <select> element has which you can use to add more interactivity. The size attribute allows you to determine how many choices are shown on screen at the same time. This is particularly helpful if you have a small number of selections as it saves the user a click. By default this value is 1 unless the multiple attribute is higher than 1, in which case it defaults to 4. The <select> element only behaves as a drop down menu when it has a value less than 2.

As mentioned above, there is also the multiple attribute. This lets you determine how many <option>s the user is allowed to select. By default this value is 1.

Accessibility

The <select> element can be used entirely with either the mouse or the keyboard.

With a Mouse: It's fairly straight forward with a mouse. You click to open and click to select if only a single choice is allowed. If multiple choices are allowed, then you can either hold CTRL while clicking to select another choice or hold SHIFT while clicking to select all of the choices between your first and newest selection. When the list has a multiple value greater than 1, to deselect an option, simply hold CTRL when clicking the option you have previously selected.

With a Keyboard: The controls are remarkably similar to the mouse with only a slight variation. Once you have tabbed over to select the <select> element with the TAB key, you simply use the arrow keys up or down to select the <option> you would like. Similarly, you can use the CTRL and SHIFT keys alongside the arrows keys if multiple selections are allowed. The space key is used to select the option.

Internationalization

As far as internationalization goes for the <select> element, it doesn't have anything extraordinary going on outside of the global attributes. The main global attributes it (and every other element) has for internationlization are dir and lang. The attribute dir takes in the value of either ltr or rtl meaning "Left to Right" or "Right to Left". As the values imply, this indicates the direction the text is to be read.

The lang attribute simply tells the computer what language the <select> element is so it does not have to guess. This is helpful for translation services as well as SEO.

Misuse

It is not that common anymore, but in many older web designs the <select> element was used as a navigation menu, with each <option> being a different page. When the user selected an <option>, it acted as a link to direct to the new page. This use case has two major faults: First, it does not semantically make sense. Although you can wrap it in a <nav> element, the fact remains that the <select> element is behaving as an <a> element. If the intended affect is for the element to behave like an <a> element, then an <a> element should be used. The second fault is that this method requires JavaScript to function. Without JavaScript, you would have no way of linking to any other page. Not only would this be terrible for SEO since bots would not be able to traverse your site, it makes your website brittle since it does not have a fallback.
