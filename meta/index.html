Meta

General Syntax & Usage

According to the MDN Web Docs: The HTML <meta> element represents metadata that cannot be represented by other HTML meta-related elements, like <base>, <link>, <script>, <style>, or <title>.

There are four different main functions of the <meta> element:

  • Providing document-level metadata. As the name implies, this is metadata that applies to the document as a whole, and is applied when the name attribute is set.
  • Providing a pragma directive, which is just a fancy way to tell the browser how it should handle the document. This is enabled when the http-equiv attribute is set.
  • Providing a charset declaration, which just tells the browser which character set to use for the document. Without this the characters might not render properly on the user's screen. This is enabled when the charset attribute is set.
  • Providing user-defined metadata, which is a catch-all for any other bits of metadata that a user would like to include. This is enabled when the itemprop attribute is set.

A <meta> element with a charset attribute should always be included on a webpage as without the correct characterset your webpage textual content might break.

A <meta> element must serve exactly one of these purposes, so you cannot have both name and http-equiv on the same element, nor can you have none of the above attributes. Luckily, you can include as many <meta> elements as you like. Here are some examples of each of those attributes:

<meta name="keywords" content="meta,metadata,data,element,HTML">
<meta http-equiv="content-type" content="text/html">
<meta charset="utf-8">
<meta itemprop="Name" content="Value">

The itemprop attribute allows you to create a name/value pair to store any data you would like to retrieve later. In any other element this attribute normally takes the text inside the element as its value, but since the <meta> element does not take a closing tag, the value is stored in the content attribute.

Interactivity

Among various keywords that the name attribute can take on, the value of viewport is important to include in one of your webpage's <meta> elements. When a website does not have a mobile version available (or is poorly optimized for mobile devices), the rendered page will often be much larger than the device and then shrunken down so that the device just receives the desktop version instead of breaking the website layout. While this is handy for older non-optimized sites, it is actively harmful if you actually do have a mobile version. To prevent this from happening you simply set the content attribute of the <meta name="viewport"> element to width=device-width, initial-scale=1. This does two things: It prevents the rendered width from being the desktop width and forces it to be only as wide as the device, and it sets the zoom scale to be 1 so the page is neither zoomed in nor zoomed out. Using it is as follows:

<meta name="viewport" content="width=device-width, initial-scale=1">

Accessibility

The first step in accessibility is having users access your website in the first place, and 99% of the time that is going to be from a search engine. While many, many things affect the SEO score of your website, two of the most critical are the <title> and <meta name="description"> elements. When you use a search engine, there are two main parts of a web result: The link and the description underneath the link. The title of that link comes from the <title> element (which is part of the reason it's technically the only required HTML element), while the short description underneath that link comes from the <meta name="description"> element.

For example, for this result for the query "YouTube", the <title> and <meta name="description"> would look like this:

<title>YouTube</title>

<meta name="description" content="Enjoy the videos and music you love, upload original content, and share it all with friends, family, and the world on YouTube.">

Internationalization

Sometimes you might not trust automatic translations of the meta data on your page so it might be useful to include different versions of your key content in different languages. To accomplish this, the global attribute lang allows you to display what language that <meta> element contains. For example, if you had a short description of your website in a <meta> element for search engines to use, you could include many different languaged versions of it like this:

<meta name="description" lang="en" content="This is a short description">
<meta name="description" lang="sp" content="Esta es una breve descripción">
<meta name="description" lang="fr" content="Ceci est une brève description">
<meta name="description" lang="de" content="Dies ist eine kurze Beschreibung">
<meta name="description" lang="jp" content="これは短い説明です">

Misuse

As soon as search engines started to take off on the internet as the main mode of web transportation, websites started to realize the importance of having their websites get ranked higher than others in the search results. In order to boost their ranking, a common practice (that doesn't work well anymore) was to stuff as many keywords as they could in the <meta> elements for their website so their website popped up for the most number of searches. This would have the unfortunate side effect though of having their website pop up for searches that weren't even remotely relevant to its content. As search engine users started to become unhappy that their searches weren't as relevant as they had liked, the search engines changed their algorithms to disuade this practice. If a website were to have a <meta> element similar to this:

<meta name="keywords" content="paper,tree,pencil,women,steel,bread,ought,company,tape,hundred,organized,prize,branch,trick,variety,practice,successful,fresh,broke,parent,condition,discussion,purple,few,lower,force,led,aware">

but the website was only one that sold paper products, you can imagine how frustrated someone who searched for bread might be when they see that website pop up in their search.
