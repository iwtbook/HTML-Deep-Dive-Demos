Description Lists (<dl>)

Overview

The <dl> element creates a description list container. It encloses a list of one or more terms (<dt>) paired with one or more descriptions (<dd>). The values within a group are alternatives; multiple paragraphs forming part of the same value must all be given within the same <dd> element. You can also use block level elements in the definition description, such as the <p> and <ul> elements. You cannot use block level elements inside a definition term.

The permitted attributes are the global attributes.

Uses

Description lists have many different applications when it comes to using semantic markup on your site. Below are some example use cases for description lists.

The primary application of a descripiton list is for defining terms. This usually results in a one to one relationship between the <dt>s and <dd>s in your markup.

<ol>
The <ol> element creates a list of items in which the ordering of the items is important to the structure of the page.
<ul>
The <ul> element creates a list of items in which the ordering of the items is NOT important to the structure of the page.
<dl>
The <dl> element creates a list of groups of one or more terms and one or more descriptions for those terms.

Another application for descripiton lists is for listing metadata that may have more than one value per key. This usually results in a one to many relationship between the <dt>s and <dd>s in your markup.
